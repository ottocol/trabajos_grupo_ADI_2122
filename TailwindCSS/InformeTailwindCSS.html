Introducción

Tailwind CSS es una potente herramienta para el desarrollo frontend. Está dentro de la clasificación de los frameworks de diseño. Permite a los desarrolladores y diseñadores aplicar estilos a los sitios web de una manera ágil y optimizada.

Tailwind permite escribir los estilos por medio de clases que se incluyen dentro del código HTML y que afectan a un aspecto concreto, por ejemplo, el fondo de un elemento, el color del texto o simplemente el margen por la parte de arriba. Esto se conoce como "Atomic CSS", por aplicarse mediante estilos muy determinados y simples.

Por tanto, Tailwind CSS no aporta muchos componentes pero aporta una gran cantidad de clases de utilidad que combinadas ofrecen prácticamente un número ilimitado de variantes de diseño.

Características principales de Tailwind CSS

Tailwind está creado siguiendo una filosofía distinta a la de los frameworks convencionales como Bootstrap.

El principal concepto que define a Tailwind es el llamado utility first, lo que significa esto es que en lugar de crear clases alrededor de componentes (botón, panel, menú, cuadro de texto...), las clases se crean alrededor de cualidades (color amarillo, fuente negrita, texto muy grande, centrar elemento...). A cada una de estas clases se les llama clases de utilidad.

Esto conlleva una serie de ventajas que veremos ampliamente en la siguiente sección pero a primera vista se observa que el uso de Tailwind nos proporciona una personalización mucho mayor de lo habitual.

Ventajas de utilizar Tailwind CSS

Mientras que otras herramientas o frameworks como Bootstrap o Materizalize CSS tienen un enfoque orientado a componentes, encontramos en Tailwind CSS un enfoque de clases de utilidad (Utility-first workflow). Este enfoque en insertar en el HTML las clases que sean necesarias para aplicar estilo a los componentes.

Las clases de utilidad son atómicas, es decir, afectan a un estilo muy concreto, aportando un valor determinado. También obliga a escribir muchas clases de utilidad en un elemento, cargando el HTML de numerosas clases que definen el estilo. Es cierto que quiebra la separación entre contenido y presentación que se tiene con HTML y CSS y hace que se forme un código denso, pero tiene algunas ventajas importantes.

Utility-first workflow obliga a escribir muchas clases de utilidad en un elemento, cargando el HTML de numerosas clases que definen el estilo. En cierto modo quiebra la separación entre contenido y presentación que se tiene con HTML y CSS y hace que se forme un código denso, pero tiene algunas ventajas importantes:

  • Los estilos se pueden escribir sin necesidad de tocar el CSS
  • El CSS de un proyecto no aumenta cuando se realizan especializaciones de estilo para algún elemento en particular.
  • Mediante las clases de utilidad podemos aplicar estilos que solo afectarán a un contenido en determinada anchura de pantalla.
  • Las clases de utilidad se pueden personalizar, creando un tema gráfico que se acompaña durante todo el diseño.
  • Tailwind CSS permite crear componentes por medio de especificación de diversas clases de utilidad, de modo que se puede reducir el uso de las mismas en los casos en los que se juzgue oportuno.
  • El desarrollo se hace mucho más rápido.
  • Fácil integración con diferentes IDEs como IntelliJ y editores tales como Visual Studio Code (IntelliSense), donde permite la función de autocompletado.

Instalación de Tailwind CSS y primeros pasos

Para utilizar Tailwind CSS, por ejemplo con npm, el gestor de paquetería más común actualmente solo habría que instalar la librería:

npm install tailwindcss

Ahora que ya tenemos Tailwind, es hora de crear el archivo base de configuración de Tailwind, este archivo podremos modificarlo posteriormente para importar plugins, temas, etc. Para crearlo lanzaremos el siguiente comando que generará un archivo llamado tailwind.config.js.

npx tailwindcss init

Ahora creamos el archivo postcss.config.js. Una vez creado el archivo, añadimos la siguiente línea:

module.exports = { plugins: [require('tailwindcss')], };

Una vez hecho esto, crearemos un archivo llamado css/tailwind.css, y añadiremos las siguientes líneas. Con esto importamos los estilos de Tailwind y podremos usarlo en nuestro proyecto. También en este archivo podremos añadir nuestras propias clases.

@tailwind base;
@tailwind components;
@tailwind utilities;

A continuación haremos uso del CLI de Tailwind para buscar y compilar todo el código Tailwind que tengamos en nuestros ficheros:

npx tailwindcss -i ./css/tailwind.css -o public/css/styles.css --watch

Siguiendo la línea del comando anterior, destacar que podemos añadir un comando al proyecto para que cuando realicemos modificaciones siempre tengamos nuestros estilos actualizados. Abrimos el archivo package.json y añadimos la siguiente línea dentro de scripts:

"scripts": {
   "dev": "postcss css/tailwind.css -o public/css/styles.css --watch"
 },

Con estos pasos realizados ya estaríamos listos para hacer uso de Tailwind.

Ejemplos de uso de Tailwind CSS

A continuación procederemos a detallar algunos ejemplos de cómo gracias a Tailwind podemos mejorar el diseño de nuestras aplicaciones de manera sencilla y tras cada ejemplo realizaremos unos breves comentarios al respecto.

<div class="p-8 mt-2 flex items-center text-orange-300 bg-gray-800">
   Contenido del elemento...
</div>

  • p-8: altera el padding para que sea de 2rem (0.25 * 8 rem)
  • mt-2: altera el margin-top para que sea 0.5rem (0.25 * 2 rem)
  • flex: pone display a flex
  • items-center: aplica el estilo "align-items: center"
  • text-orange-300: pone el color del texto a naranja más bien claro.
  • bg-gray-800: pone el color de fondo gris muy oscuro.

A continuación el código que implementa una tarjeta:

<!-- TAILWIND CSS -->
<article class="mb-5 bg-teal-100 border border-gray-200 rounded shadow-md">
<img src="..." alt="..." class="rounded rounded-b-none">
<div class="p-3">
 <h3 class="mb-1 text-xl font-bold">Hola Mundo!!</h3>
 <p class="mb-4">Que bonito es Tailwind CSS</p>
 <a class="bg-orange-600 text-white px-4 py-2 font-bold uppercase rounded-xl tracking-wider" href="#">Primary</a>
</div>
</article>

Hemos generado una tarjeta a base de incorporar algunas clases. Aunque se vea el HTML más cargado que un HTML clásico obtenemos ventajas gracias a las clases de utilidad. Haciendo uso de esto hemos creado un componente de tarjeta completamente personalizable algo que por ejemplo en Bootstrap no podríamos hacer sin añadir aparte clases CSS de manera separada. Simplemente aplicando alguna clase de utilidad diferente podemos cambiar por ejemplo el borde redondeado para hacerlo más o menos sutil, el sombreado de la tarjeta, etc.

Ahora veremos un ejemplo de un botón que cambia de color al pasar por encima con el ratón:

<button class="bg-blue-500 hover:bg-blue-400 text-white font-bold py-2 px-4 border-b-4 border-blue-700 hover:border-blue-500 rounded">
 Botón
</button>

Otro ejemplo de código sería el siguiente:

<div class="text-center p-2">
 <div class="inline-flex items-center bg-white rounded-full p-2 shadow text-sm">
   <span class="inline-flex bg-pink-600 text-white rounded-full h-6 px-3 justify-center items-center">Pink</span>
   <span class="inline-flex text-pink-600 px-2">Botón rosa</span>
 </div>
</div>

La explicación de algunas clases es la siguiente:

  • text-center: para centrar todo los elementos que aparezcan después.
  • p-2: padding 2
  • inline-flex: Para limitar el tamaño del contenedor
  • bg-pink-60: Establece el color rosa fuerte
  • text-white: color de la letra en blanco
  • rounded-full: clase para redondear bordes
  • h-6: altura 6
  • px-2: padding izquierdo y derecho
  • shadow: Añade sobra por debajo y los lados

Conclusiones

Como hemos visto a lo largo de este informe el uso de Tailwind nos aporta mucho pero puede suponer un código ligeramente más sucio y denso al tener que añadir todo el css mediante clases de utilidad.

Por tanto podemos concluir que el uso de Tailwind será positivo en función del tipo de proyecto que estemos desarrollando, si deseamos un proyecto con un diseño avanzado y una fácil y fuerte personalización de los  componentes de la IU este framework nos aportará mucho, sin embargo, si el diseño de nuestra aplicación es algo que pasa a segundo plano sería conveniente usar otro tipo de librerías como Bootstrap que proporciona clases predefinidas para gran cantidad de componentes.
